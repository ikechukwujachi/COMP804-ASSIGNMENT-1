Web Attacks Information

Name Web Attacks
XSS (Cross-Site Scripting) XSS involves injecting malicious executable scripts into trusted applications or websites. Attackers typically initiate an XSS attack by sending malicious links to users, enticing them to click. Without proper data sanitisation, these links execute the attacker's code on the user's system, allowing them to steal active session cookies.
Source - https://www.synopsys.com/glossary/what-is-cross-site-scripting.html
SQL Injection SQL Injection is one of the most common web attacks, that has the ability to destroy a database. SQL Injection is when an attacker inserts malicious code in SQL statements/queries via webpage input fields. Exploiting any vulnerabilities, in order to manipulate the database.
Source - https://www.w3schools.com/sql/sql_injection.asp
Clickjacking Also known as s "UI redress attack", Clickjacking is when an attacker uses multiple transparent/disguised elements (buttons and links) on a page to "trick" users. Thus, the attacker "hijacks" clicks, leading them to unintended actions such as making unauthorised transactions or revealing sensitive information.
Source - https://owasp.org/www-community/attacks/Clickjacking
Phishing Phsising is an attempt to steal sensitive information. Attackers typically "phis" for usernames, passwords and credit card information, that is utilised or sold. The attackers pose as a reputable source, enticing users. Similarly, to fishermen using bait to catch fish.
Source - https://www.cloudflare.com/learning/access-management/phishing-attack/
Brute Force Attack A brute force attack is a hacking technique where the hacker repeatedly tries different combinations of passwords, usernames, or encryption keys until they find the right one. This method, though basic, is quite effective for breaking into individual accounts or the systems and networks of organisations without permission.
Source - https://www.fortinet.com/resources/cyberglossary/brute-force-attack
Session Hijacking Session Hijacking is a cyber attack where the attacker takes over an users active session on a website, application or network services. They do this, by interceptiing or stealing the session identifier, the attacker can impersonate the user and gain access to an account or system. Allowing the attacker to perfom actions on the users behalf.
Source - https://owasp.org/www-community/attacks/Session_hijacking_attack
DNS Spoofing DNS spoofing is when a hacker messes with a DNS server to send users to a fake website they control. This sneaky trick can happen on public Wi-Fi or anywhere the hacker can mess with the network settings to redirect users to their malicious site, putting users at risk of scams or data theft.
Source - https://www.proofpoint.com/au/threat-reference/dns-spoofing
Remote Code Execution (RCE) A remote code execution (RCE) assault occurs when an adversary gains the capability to execute harmful code on a target organisation's computers or network. This capability allows the attacker to perform various nefarious activities, such as deploying supplementary malware or pilfering sensitive data.
Source - https://www.cloudflare.com/learning/security/what-is-remote-code-execution/
Drive-by Download Attack A drive-by download attack occurs when malicious code is inadvertently downloaded to your computer or mobile device, making you vulnerable to cyberattacks. Unlike traditional infection methods, such as clicking on links or opening malicious attachments, drive-by downloads require no user interaction like clicking or downloading. These attacks exploit security vulnerabilities in applications, operating systems, or web browsers, often due to outdated or incomplete updates.
Source - https://www.kaspersky.com/resource-center/definitions/drive-by-download
Man-In-The-Middle (MITM) A man-in-the-middle (MITM) attack occurs when an attacker intercepts communication between a user and an application, aiming to eavesdrop or impersonate one of the parties. The goal is to steal personal information such as login credentials and credit card details, with targets often including users of financial apps, e-commerce sites, and platforms requiring authentication.
Source - https://www.imperva.com/learn/application-security/man-in-the-middle-attack-mitm/
